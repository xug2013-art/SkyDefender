# 游戏美术资源生成技能

为《星空入侵者》游戏生成符合项目规范的像素风格美术资源，自动优化提示词生成符合要求的图片。

## 前置依赖
1. 需要先安装 `minimax-multimodal-toolkit` 技能才能使用
2. 提前配置环境变量：
   - `MINIMAX_API_KEY`: MiniMax API密钥（在https://platform.minimaxi.com/获取）
   - `MINIMAX_API_HOST`: 固定为`https://api.minimaxi.com`（中国大陆）或`https://api.minimax.io`（全球）
3. SKILL启动时会自动校验环境变量是否配置，无需手动检查
4. 【重要】风格校验使用Claude内置多模态能力完成，无需安装或调用任何外部视觉分析类工具

## 使用方法
当用户描述需要生成游戏相关美术资源时，自动调用此技能，用户也可以手动触发：
```
/skill game-asset-generator "你的资源描述"
```

## 示例
用户输入："帮我生成一个小型红色敌机"
自动调用技能，生成对应资源

用户输入："/game-asset-generator 紫色加血道具，漂浮效果，心形像素图标"
直接生成加血道具资源

其他触发场景：
- "生成第二关Boss"
- "做一个子弹的美术资源"
- "需要一个蓝色护盾道具图片"
- "生成游戏背景图"

## 特性
✅ 自动优化为英文提示词，提高生成准确率
✅ 自动适配API尺寸限制，100%对齐用户输入的尺寸/比例要求，不做自动假设
✅ 生成纯白背景的像素风格图片，方便后续手动处理
✅ 生成文件保存在 minimax-output/ 目录下
✅ 自动记录生成信息，方便后续处理
✅ 自动匹配《资源需求清单.md》中的标准描述，无需用户输入完整提示词
✅ 内置多模态校验闭环，不合格结果自动优化重试，无需人工干预

## 执行逻辑
1. **参数接收**：接收用户的资源描述、尺寸/比例要求作为输入，可选读取`资源需求清单.md`作为上下文参考，匹配对应资源的完整官方提示词，无匹配则使用用户输入的描述。
2. **提示词优化**：
   - **翻译标准化**：所有中文描述自动翻译为专业英文游戏美术提示词，禁止中英文混合输入，避免语义理解偏差
   - **通用规范拼接**：固定追加像素风格通用约束：`16-bit pixel art, white background, no shadows, no effects, clean pixel art, no extra details, fit for WeChat mini game`
   - **视角动态适配**：仅当资源类型为敌机/玩家/子弹/战机/Boss等飞行单位时，自动追加`top-down orthographic view, head facing straight down`正俯视视角约束，其他类型资源不添加
   - **核心需求加权**：对用户明确提出的核心特征（如造型、配色、特殊结构）添加权重标记，格式为`(关键词:权重值)`，权重值1.2-1.5，提高核心特征的生成优先级
   - **重试定向优化**：校验不通过时，仅针对不符合的需求点调整提示词，追加针对性描述（如不符合蜻蜓造型则追加`definitely dragonfly shape, not other insects`），不全局修改原有提示词，避免其他合格特征丢失
3. **尺寸适配**：根据用户输入的尺寸/比例要求，等比例换算到MiniMax API支持的尺寸范围（512px-2048px，8的倍数），不做任何自动比例假设。
4. **文件命名**：自动将用户描述转换为小写下划线格式的文件名，带版本号区分重试版本。
5. **图片生成**：调用MiniMax多模态工具包生成适配后的尺寸图片。
6. **风格校验**：
   - 【明确】使用Claude内置多模态能力（无需调用外部工具），直接读取生成的实际图片文件，从专业美术设计师视角进行实际校验
   - 校验维度：逐条核对是否符合用户所有明确需求点（造型、配色、风格、视角、像素效果等），忽略透明背景相关要求（因MiniMax API不支持直接生成透明背景）
   - 校验通过：进入结果输出环节
   - 校验不通过：自动提取不符合点，定向微调prompt关键词（仅修改不符合部分，不全局调整），回到图片生成环节重新生成，最多重试2次，避免无限循环
   - 【异常兜底】若因任何原因（如图片读取失败、多模态能力不可用等）无法完成自动校验，必须明确告知用户当前校验状态，列出所有待校验的需求点，由用户确认是否符合要求，禁止直接交付未校验的结果
7. **结果输出**：
   - 原始生成版（高清大图）：校验通过后保存到`minimax-output/`目录，文件名与资源名称对应
   - 自动处理流程：使用FFmpeg自动完成透明背景去除+目标尺寸适配，具体命令为：
     ```bash
     ffmpeg -i minimax-output/原始文件名.png -filter_complex "[0:v]colorkey=white:0.1:0.1,scale=目标宽度:目标高度" images/最终文件名.png -y
     ```
   - 最终交付版（适配尺寸+透明背景）：处理完成后保存到项目`images/`目录，替换对应占位文件
   - 原始高清版备份：同步将原始生成版备份到`images_large/`目录，用于后续调整修改
   - 输出完成后提示用户验证效果。

## API规格说明
1. **尺寸限制**：MiniMax image-01模型支持的生成尺寸范围为**512px-2048px**，且必须为8的倍数。对于用户要求的小于512px的尺寸需求，SKILL会自动生成最接近的API支持尺寸的高清原图。
2. **透明背景限制**：MiniMax image-01模型目前不支持直接生成带透明alpha通道的PNG图片，默认生成纯白背景的图片，需要用户后续自行处理透明背景。
